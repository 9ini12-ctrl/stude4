# منصة رخصة ممارس

نسخة كاملة جاهزة للنشر على **GitHub + Netlify** مع **Neon PostgreSQL**، مبنية وفق الـ PRD المرفق، وتشمل:

- تسجيل الدخول
- إنشاء أول مدير للنظام لأول مرة
- إدارة المستخدمين
- إدارة المعلمين والمعلمات
- فصل كامل بين الرجال والنساء
- واجهة للمشرف/المشرفة
- واجهة للمقرئ/المقرئة
- تقارير عامة وخاصة
- واجهة عامة بدون تسجيل دخول
- حالات Loading / Empty / Error
- Progress Bars + Circular Progress
- Netlify Functions + PostgreSQL

---

## 1) المتطلبات

- حساب Netlify
- مستودع GitHub
- قاعدة بيانات Neon PostgreSQL
- متغيرات البيئة:
  - `DATABASE_URL`
  - `JWT_SECRET`

---

## 2) إعداد قاعدة البيانات

نفّذ ملف:

```sql
sql/01_schema.sql
```

إذا أردت **مسح جميع البيانات القديمة** مع الإبقاء على الجداول، نفّذ:

```sql
sql/02_reset.sql
```

---

## 3) متغيرات البيئة

أضف في Netlify أو ملف `.env` أثناء التطوير:

```env
DATABASE_URL=postgres://USER:PASSWORD@HOST/DB?sslmode=require
JWT_SECRET=your-super-secret-key
JWT_EXPIRES_IN=7d
```

---

## 4) التشغيل المحلي

```bash
npm install
netlify dev
```

أو لعرض الواجهات الثابتة فقط:

```bash
python3 -m http.server 8080
```

> ملاحظة: الواجهات ستحتاج البيئة الخلفية وقاعدة البيانات حتى تعمل الوظائف بالكامل.

---

## 5) أول تشغيل للنظام

عند فتح الصفحة الرئيسية لأول مرة:
- إذا لم يوجد أي مستخدم في النظام، ستظهر لك شاشة **إنشاء أول مدير**
- بعد الإنشاء، تستطيع تسجيل الدخول مباشرة

## 7) ملاحظات النظام

### الأدوار
- `admin`
- `supervisor` + `male/female`
- `reader` + `male/female`

### قواعد الفصل
- المشرف الرجل يدير المعلمين فقط
- المشرفة تدير المعلمات فقط
- المقرئ يرى المعلمين المربوطين به فقط
- المقرئة ترى المعلمات المربوطات بها فقط
- المجاز / المجازة لا يظهر للمقرئ / المقرئة

### حساب نسبة الأجزاء
- الرجال: من 30
- النساء: تعرض 30 جزءًا لكن النسبة تحسب من 15 فقط وبحد أعلى 100٪
- المجاز / المجازة = 100٪ مباشرة

### الأوزان النهائية
- الأجزاء: 40%
- الاختبارات: 25%
- المهام: 20%
- النهائي: 15%
